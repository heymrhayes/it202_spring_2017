Activity: Add User Filter to the NYT Article Search

In this activity, we'll update the NYT article search page to let the user enter a search term.

We'll update the script to run in response to a button click event rather than on page load. When the button is clicked, we'll read the value of the input box and use that to construct the URL for the AJAX call.

Instructions

Part 1 - Edit the HTML

  1. Open either your version or mine (https://heymrhayes.github.io/it202_spring_2017/examples/20170201.html, and save the content in a new file.
  2. Add a text input box; give it an id attribute with a value of txtQuery.
  3. Add a button; give it an idid attribute with a value of btnSearch.

Part 2 - Run on button click

  1. Inside the jQuery DOM ready function, add a click event handler for the button. Create the anonymous function; note where it ends.
  2. Move the entire script that previously ran at page load into the anonymous function you just created.
  3. Test and verify that you're getting the same article set and display as before; the only difference is that it should happen in response to a button click.

Part 3 - Get the search term and update the URL

  1. The jQuery val() function returns the value(s) for the specified element(s). In the first line of the click event handler, read the value of the input box and store it in a variable named query.
  2. Wherever you're constructing the URL, replace any hard-coded value with the variable.
  3. Test. What happens when you make subsequent searches?

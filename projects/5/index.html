Project 5: 3-screen app for client

In this project, you'll have the experience of building a product for someone that meets their needs. The purpose is for you to go through the process: determining requirements, creating the first version, testing and getting feedback from the client, then revising and delivering. The amazingness of the app is not what this project is about.

Your file should be /projects/project5.html. Your app should have the following features:

  1. Three screens with different content. The content can be static (pictures of you, information they need to retrieve frequently, etc.) but the data should be different on each screen. (You can certainly choose to pull and present data from the world, but that's not necessary.);
  2. Navigation that lets the user easily get to each screen.
  3. Uses the MDLite framework. Take a look at the layout components; they will handle most of the app for you.
  4. Includes a manifest and the meta tags necessary for it to be loaded on the device as an app (refer to the airhorner lab);
  5. An additional screen ("About") with information about the developer.

Instructions

  1. Choose a client - a relative or friend.
  2. Ask them what information they look up frequently. If you know the person well, you may have a good idea of what they would find useful.
  3. Choose an MDLite layout component (not template) and enter content.
  4. Create a manifest.json file and add the meta tags necessary.
  5. Test with the client. Document the feedback you get.
  6. Revise and test your app, repeating as needed.
  7. Deliver.

Evaluation

40% for functionality (does it function, not "is it amazing"

30% for visual appearance and completeness

30% for documented testing and feedback

Again, these can be screens of static content, but they should serve a purpose for the client.

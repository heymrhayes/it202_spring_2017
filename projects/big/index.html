The Big Project: a feature-rich, functional app

In this project, you'll demonstrate your ability to combine a number of the languages, tools and features we've covered this semester into a usable app.

My intent is that you have a project you can be proud of and that says something about your capabilities to a potential employer and/or to other developers. And in my ideal world, you create an app that is actually useful to you and/or others.

Requirements

Your app must include the following features:

  • It must work well on a mobile device.
  • It must be a Single Page App (not a set of pages.)
  • It must have at least 3 screens, plus an About screen.
  • It must have clear navigation.
  • It must use the MDLite framework.
  • It must use the jQuery library.
  • It must be installable on the phone as an app (manifest.json and the required meta tags)
  • It must utilize a Service Worker and the Cache Storage API to cache and deliver assets.

    (Note that all of the above can be accomplished without knowing what data you'll use. In fact, a template with all those features would be good to have as a starting point for lots of apps you might build.)

  • It must utilize data pulled from an API; at least one data call must be in response to user action (e.g, a search or selection.)
  • It must utilize the Indexed DB API (via a wrapper like Dexie.js) to store and retrieve data.
  • It must include some visualization (map, chart, something).
  • It must include one or more images.

In addition to the app itself, your project must:

  • Include an early review with me (can be done via email if we can't find a meeting time);
  • Show evidence of planning (a document or the use of issues in GitHub);
  • Include a document with the results of some user testing;
  • Use good naming conventions for variables, functions, etc in your code;
  • Use good commenting in your code;
  • Provide commented attribution for code you found elsewhere (with url of the source);

I reserve the right to add one or two things depending on what additional topics we cover.

So what might this look like?

You use apps all the time; take a look at a few of them and see how they function. How many screens? What's on each one?

Your app could retrieve, display and manipulate data; it could be a game; it could allow creation of data; it could aggregate data from a variety of sources; it could be a drawing app (using the canvas element); or something much more creative than I can imagine.

You've been building up to this all semester, and this project requires that you assemble capabilities from prior projects and labs into a coherent whole.

Planning will be important to completing the project on time and with minimal stress, and you can get a start on large chunks of the project even if you haven't decided on the data you'll be working with. (If you don't have any ideas, I can provide a couple.)

Evaluation

I will provide some numbers on the individual items, but this should look good and function well.

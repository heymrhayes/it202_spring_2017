Project 3: Simple Scrolling Game

In this project, you'll demonstrate knowledge of JavaScript and the HTML5 Canvas element by making a simple scrolling game.

Your file should be /projects/project3.html. Your game should have the following features:

  1. A "Welcome" screen with a game name and instructions. When the user clicks the canvas, the game displays and starts.
  2. A player character on the left side of the screen that moves only vertically and which the player controls using the up/down arrow keys;
  3. "Harm" characters which move from right to left and which, upon collision with the player, result in a loss the player's lives;
  4. "Benefit" characters which also move from right to left and which, upon collision with the player, adds points to the score;
  5. Display of lives left and score on the top of the screen;
  6. A "Game Over" screen when the player has used their last life.
  7. One type of character (player, harm or benefit) must be an image.

Hints/Suggestions

  1. To start, get a single item to move from right to left across your canvas.
  2. When it gets to the left edge, move it back to right side and set it's y position randomly.
  3. Create an array of these items, using a constructor to create each one; use a parameter for type of character (harm or benefit.)
  4. Get a player character to move up and down in response to arrow keys.
  5. Create and display (with labels) variables for score and lives left.
  6. Get collision testing working; upon collision, check the type of the object that collided, and update the variables accordingly.
  7. Upon collision, display a screen with a relevant message and continue the game when the user clicks the canvas.
  8. When the lives left are negative, display a Game Over message.
  9. Get started early.
  10. Ask questions on Piazza (I don't expect you to have all the answers.)
  11. Test step by step.
  12. Save and push to your GitHub repo.

Evaluation

25% for characters, display text and animation

25% for collision handling

25% for message screens (start, collision, game over)

25% for visual appearance and creativity

Extra points for parallax scrolling and/or device orientation control (on a phone or tablet)

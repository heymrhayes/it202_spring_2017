Lab 3: First Bootstrap Page

Bootstrap is indeed "the most popular HTML, CSS, and JS framework for developing responsive, mobile first projects on the web", and you'll want to be familiar with the basic concepts so that you can create your own layouts and templates and/or manipulate others' layouts and templates.

In this lab, you'll download download a template, upload it to Cloud9, make some modifications, update your Git and push it up to GitHub. You'll also enable GitHub pages on your repo so that you and others can access your pages.

The accomplishment we're going for is that you understand how to use and manipulate a Bootstrap template, how to diagnose and resolve file references in a page and that you practice with c9, Git and GitHub.

Materials List

  • Your class Cloud9 workspace
  • A folder in that workspace named labs
  • Your GitHub account

Instructions

Part 1 - Copy a Bootstrap template

  1. In your labs folder, create a new empty file (no template.)
  2. Navigate to the Bootstrap examples on the Getting Started page at http://getbootstrap.com/getting-started/.
  3. Open the Jumbotron example, view and copy the page source, and paste the contents to your c9 file. Note that there is no color-coding, text-completion or any other feature; in the absence of a file extension, the editor doesn't know what color-coding or other features make sense for the file.
  4. Save your file as lab3.html.
  5. Change the content for both the main headline and the title element to "Lab 3: LastName, First Name" (e.g. Lab 3: Hayes, Dave).
  6. Save your file again.
  7. Preview it (Preview > Live Preview File), pop it to a new window, and note that it doesn't look like it should.

Part 2 - Diagnose the problem(s)

  1. In the tab with your previewed file, open the Dev Tools to the Network tab, and refresh the page. Make sure you're displaying all file types.
  2. Notice the items in red with a status of 404. Open the HTTP Status Codes page linked in the Resources section, and find out what status 404 indicates.
  3. Click the first 404 item in the Network tab to open the details pane, and look at the Request URL. Note that it refers to a path and file that isn't in your c9 workspace.
  4. In your lab3.html file, find the html tag that generated that request. (Use the editor's Find feature to search for the file name.)

Part 3 - Solve the problems

The example page that we copied included relative file and path references that were valid based on the location of that file on Bootstrap's server.

We didn't copy their entire folder and file structure, we just copied the contents of one file. We have two options for dealing with this situation: 1 - create copies of the needed files on our server and adjust the URL paths in our file; or 2 - find out where the files reside on another server and use absolute URL paths in our html file to point to those locations.

Let's go with option 2.

  1. You've already found the location in your html file where the first missing file is referenced. We need to know the URL to that file on Bootstrap's server(s), and we can use the Dev Tools to find the full path. Go back to the tab that displayed their Jumbotron example, open the Dev Tools Network tab, refresh the page, and find the relevant file in the list. Click that entry to open the details pane. In the Headers pane, expand the General section to see the absolute URL to that file. Copy the address, and open it directly in a new browser tab to make sure it's correct and accessible.
  2. In your lab3.html file, in the tag that references that file, replace the existing relative reference with the absolute URL. Save your file, refresh the browser tab that's displaying your page, and check the Dev Tools Network tab. Note the status for the page now.
    Mixed Content
    Depending on what type of file you're referencing, you may see an error message about "mixed content." The issue is that your lab3.html page is being served by Cloud9 using HTTPS (secure) and the absolute reference you pasted uses HTTP (non-secure.)
    Change the protocol used in the reference to that file to HTTPS, and it should work. Some content can't be accessed using HTTPS because it's not been enabled on the server; in such a case, you'd need to make your own copy and deliver it from your server.
  3. Fix the other invalid file references using the same method.

Part 4 - Modify the template layout

  1. Add a second row of "tiles" by finding the div with the class row, and duplicating it; paste a copy of the entire "row" div below the original but still inside the parent div with the class container (and above the hr and footer elements.
  2. Add another "tile" to the first row by copying one of the existing elements and pasting it inside the "row" div. Notice that the layout is disrupted and that new tile is displayed on a page by itself.
    Why is the layout disrupted?
    Why? Because we have created four 4-column elements, or 16 total columns of content. The Bootstrap grid model is based on 12 columns of content.
  3. Change the tiles in that first row to be 3-column elements by modifying their class attributes. They should now display on one line, assuming the screen is wide enough.

Part 4 - Make and push changes, and verify.

  1. After you've made all the changes and verified that your page is working, do a final save of your file.
  2. In the terminal, run the command git status to see the differences between your file system and the master branch on GitHub. Note the message text and message color.
  3. Add file contents to your local (c9) Git by running git add *
  4. Run git status again, and again, note the message text and message color.
  5. Record those changes to your local (c9) Git by running git commit -m "intial load lab3".
  6. Push your local (c9) Git to GitHub by runninggit push origin master.
  7. Examine your repo on GitHub and verify that it shows your latest version.

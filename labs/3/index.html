Lab 3: Integrating Cloud9 and GitHub

GitHub has become an indespensible tool and platform for developers; you'll use it to share your work and to demonstrate your skills to potential collaborators and employers.

In this lab, you'll create a file, create a Git repo in your Cloud9 workspace, create an empty GitHub repo, associate your c9 workspace with that repo, and push your Cloud9 workspace Git to that GitHub repo.

The accomplishment we're going for is that you understand how to work with c9, Git and GitHub. Make sure to take some time to understand what you're doing in each step; you'll be working with these tools all semester (and possibly beyond.)

Materials List

  • A Cloud9 account
  • A c9 workspace named it202
  • A folder in that workspace named labs
  • A GitHub account

Instructions

Part 1 - Create a new folder and file in your c9 folder

  1. In your labs folder, create a new file named lab3.html. Use the c9 default html template.
  2. Add an h1 element with the content "Lab 3: LastName, First Name" (e.g. Lab 3: Hayes, Dave) and a title element with the same content.
  3. Save your file.

Part 2 - Create a new, empty GitHub repo

  1. Navigate to GitHub and log in to your account.
  2. Create a new repo, name it it202, and leave the "Intialize this repository..." box unchecked. (You will be importing an existing repository.)
  3. When the repo opens, get the URL for the repo by clicking the green Clone or download button; copy the URL for use in the next part.

Part 3 - Associate your c9 workspace to your GitHub repo and push your local repo

  1. In your c9 IDE, navigate to the bash terminal window; if it's not already open, select Window > New Terminal from the menu.
  2. Make sure your command prompt shows you're at the top level of your directory, because you're going to create one repo for all (or at least most) of your coursework. The prompt should show <your username>:~/workspace. If it doesn't, close that window and open a new terminal window.
  3. Follow the instructions at https://help.github.com/articles/adding-an-existing-project-to-github-using-the-command-line/#platform-linux to set your new GitHub repo as the master for your c9 repo.
  4. Look at your repo on GitHub and verify that your file(s) have been pushed.

Part 4 - Make and push changes, and verify.

  1. In c9, open your lab3.html file, and add a p element below the existing h1 element, with the content Hello, World. Save the file.
  2. In the terminal, run the command git status to see the differences between your file system and the master branch on GitHub. Note the message text and message color.
  3. Add file contents to your local (c9) Git by running git add *
  4. Run git status again, and again, note the message text and message color.
  5. Record those changes to your local (c9) Git by running git commit -m "update lab3".
  6. Push your local (c9) Git to GitHub by runninggit push origin master.
  7. Examine your repo on GitHub and verify that it shows your latest version.
